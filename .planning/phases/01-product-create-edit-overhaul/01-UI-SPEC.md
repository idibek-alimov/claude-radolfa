## Spacing Scale

Declared values (multiples of 4 only). Project uses Tailwind utility classes mapped to this scale.

| Token | Value | Tailwind Class | Usage |
|-------|-------|---------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps, inline button padding |
| sm | 8px | `gap-2`, `p-2` | Compact element spacing, toolbar gaps |
| md | 16px | `gap-4`, `p-4` | Default element spacing, form field gaps |
| lg | 24px | `gap-6`, `p-6` | Card internal padding (matches existing `p-6` in edit page cards) |
| xl | 32px | `gap-8`, `px-8` | Column gap in two-column layout |
| 2xl | 48px | `py-12` | Page vertical padding |
| 3xl | 64px | `py-16` | Not used in this phase |

Exceptions:
- Touch targets (interactive icon buttons): minimum 28px hit area via `h-7 w-7` (consistent with existing SKU table rows in `CreateProductDialog`)
- Card container outer padding: `px-4 sm:px-6 lg:px-8` (matches `ProductEditPage.tsx` pattern)
- Section dividers inside cards: `border-t pt-4` (existing pattern from `CreateProductDialog`)

## Color

All colors use CSS variables defined in `globals.css`. Values below are the resolved HSL approximations from the `:root` block.

| Role | CSS Variable | Resolved Approx | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `hsl(var(--background))` | #ffffff | Page background, card backgrounds (`bg-card` = white) |
| Secondary (30%) | `hsl(var(--muted))` / `hsl(var(--secondary))` | #f5f5f5 | Page wrapper `bg-muted/30`, table header `bg-muted/50`, section dividers |
| Accent (10%) | `hsl(var(--primary))` = 239 84% 67% | #5b6ef5 (violet-blue) | Primary action button only тАФ "Create Product" submit button |
| Destructive | `hsl(var(--destructive))` = 0 84.2% 60.2% | #f04343 (red) | Inline validation error text, SKU row delete icon on hover |

Accent (`--primary`) reserved for: **submit button only** ("Create Product", "Creating...", "Uploading images..." states). All secondary actions (Cancel, Add Size, navigation links) use `variant="outline"` or default text styling тАФ never primary color.

Border color: `hsl(var(--border))` = #e5e5e5 тАФ used on all cards, inputs, and the TipTap editor wrapper.

## Layout Contract

### Create Page (`/manage/products/new`)

Container: `min-h-screen bg-muted/30 py-10`
Inner wrapper: `max-w-[1600px] mx-auto px-4 sm:px-6 lg:px-8` (source: D-05)

Two-column grid inside the card area:
- Left column (approx 55% width): Name field, Category select, Color select, Rich text description with char counter
- Right column (approx 45% width): Image upload zone + SKU table
- Grid class: `grid grid-cols-1 lg:grid-cols-[1fr_auto] gap-8` тАФ single column on mobile, two on desktop
- Admin-only context: mobile layout not required (REQUIREMENTS.md: "Admin is desktop-first")

Page structure top-to-bottom:
1. Breadcrumb: `<ChevronLeft> Back to Products` link тАФ `text-sm text-muted-foreground` (matches `ProductEditPage`)
2. Page heading: "Create Product" тАФ `text-2xl font-bold text-foreground`
3. Subheading (optional): "Fill in the details below to add a new product to the catalog." тАФ `text-sm text-muted-foreground mt-1`
4. Two-column form area (one card wrapping both columns OR two separate cards тАФ left card + right card)
5. Form action bar: Cancel + Create Product buttons тАФ `flex justify-end gap-3 pt-4`

### Edit Page (`ProductEditPage.tsx`)

Single CSS change: `max-w-3xl` тЖТ `max-w-[1600px]` on line 48. All other layout elements unchanged (source: D-10).

## Rich Text Editor Contract (`RichTextEditor`)

**Library:** TipTap v3 with `@tiptap/starter-kit` (source: D-01, RESEARCH.md)

Toolbar buttons (source: CONTEXT.md "Claude's Discretion" тАФ standard set):
| Button | Action | Active State |
|--------|--------|-------------|
| **B** | toggleBold | `data-[active=true]:bg-muted rounded` |
| *I* | toggleItalic | same |
| Unordered list | toggleBulletList | same |
| Ordered list | toggleOrderedList | same |

Toolbar container: `flex gap-1 border-b border-input px-2 py-1.5`
Editor content area: `px-3 py-2 min-h-[120px] text-sm` (increased from 100px to give usable writing area)
Outer wrapper: `rounded-md border border-input bg-transparent text-sm shadow-sm focus-within:ring-1 focus-within:ring-ring`

Character counter: displayed below editor тАФ `text-xs text-muted-foreground text-right`
- Count method: `editor.getText().length` (plain text, not HTML length) тАФ source: RESEARCH.md open question, resolved to plain text
- Limit: 5000 characters (source: D-03)
- Format: `{count} / 5000`
- Warning state at 4500+: counter turns `text-destructive`

## Submit Button State Machine

Button is the primary visual indicator of async progress (source: D-09).

| State | Label | Icon | Disabled |
|-------|-------|------|---------|
| idle | "Create Product" | none | false |
| creating | "Creating..." | `Loader2 animate-spin h-4 w-4 mr-1.5` | true |
| uploading | "Uploading images..." | `Loader2 animate-spin h-4 w-4 mr-1.5` | true |

No separate progress bar. No step indicator. Spinner inside the button is the only feedback.

## Image Upload Zone Contract

Interaction (source: D-07):
- Hidden `<input type="file" multiple accept="image/*">` triggered by a button click
- Button label: "Add Images" with `ImagePlus` icon (`lucide-react`)
- Selected files shown as thumbnail previews using `URL.createObjectURL(file)`
- Thumbnail size: 64px ├Ч 64px (`h-16 w-16`), rounded (`rounded-md`), `object-cover`
- Each thumbnail has an `X` remove button (top-right overlay): `h-4 w-4 text-white bg-black/50 rounded-full`
- No files selected state: show a dashed bordered zone with "Add Images (optional)" text in `text-sm text-muted-foreground`

Upload is optional (source: CREATE-04). No validation error is shown when no images are provided.
