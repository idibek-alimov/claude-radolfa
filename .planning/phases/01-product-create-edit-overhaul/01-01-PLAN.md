       Run: `npm install @tiptap/react @tiptap/starter-kit --prefix frontend`

    2. **Create `frontend/src/shared/ui/RichTextEditor.tsx`**:
       Build a reusable client component wrapping TipTap with the following spec (from UI-SPEC):

       ```tsx
       "use client";

       import { useEditor, EditorContent } from "@tiptap/react";
       import StarterKit from "@tiptap/starter-kit";
       ```

       **Props interface:**
       ```typescript
       interface RichTextEditorProps {
         initialContent: string;
         onChange: (html: string) => void;
         maxLength?: number;  // default 5000 (per D-03)
       }
       ```

       IMPORTANT: Use `initialContent` (not `content`) to make clear this is only the initial value. TipTap content must NOT be reactively updated after mount (Pitfall 2 from RESEARCH.md).

       **Toolbar** (4 buttons, per UI-SPEC):
       - Bold (B) — `editor.chain().focus().toggleBold().run()`
       - Italic (I) — `editor.chain().focus().toggleItalic().run()`
       - Bullet list (list icon from lucide-react: `List`) — `editor.chain().focus().toggleBulletList().run()`
       - Ordered list (ordered list icon from lucide-react: `ListOrdered`) — `editor.chain().focus().toggleOrderedList().run()`

       Each toolbar button:
       - `type="button"` (prevent form submission)
       - `aria-label` per UI-SPEC: "Bold", "Italic", "Bullet list", "Ordered list"
       - Active state: `data-[active=true]:bg-muted rounded` (use `data-active={editor?.isActive("bold")}` pattern)
       - Styling: `px-2 py-1 rounded data-[active=true]:bg-muted`

       **Toolbar container:** `flex gap-1 border-b border-input px-2 py-1`

       **Editor content area:** `<EditorContent editor={editor} className="px-3 py-2 min-h-[120px] text-sm" />`

       **Outer wrapper:** `rounded-md border border-input bg-transparent text-sm shadow-sm focus-within:ring-1 focus-within:ring-ring`

       **Character counter** below the editor (per D-03, UI-SPEC):
       - Use `editor.getText().length` for count (plain text, not HTML — per RESEARCH.md recommendation)
       - Format: `{count} / {maxLength}`
       - Normal state: `text-xs text-muted-foreground text-right`
       - Warning state (count >= maxLength - 500, i.e. 4500+): `text-xs text-destructive text-right`
       - In `onUpdate`, if `editor.getText().length > maxLength`, call `editor.commands.undo()` to reject the input. This enforces the limit without truncating HTML.

       **Export:** `export function RichTextEditor({ initialContent, onChange, maxLength = 5000 }: RichTextEditorProps)`

       **useEditor config:**
       ```tsx
       const editor = useEditor({
         extensions: [StarterKit],
         content: initialContent,
         onUpdate: ({ editor }) => {
           if (maxLength && editor.getText().length > maxLength) {
             editor.commands.undo();
             return;
           }
           onChange(editor.getHTML());
         },
       });
       ```

       Do NOT pass `initialContent` as a reactive dependency to useEditor. Only use it for the initial `content` value.
  </action>
